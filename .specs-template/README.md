# Шаблон структуры .specs/ для приложений

**Версия**: 1.0  
**Дата**: 2025-11-27

## Назначение

Эта папка содержит шаблон структуры `.specs/` для всех приложений в NX монорепозитории. Каждое приложение должно иметь свою папку `.specs/` для хранения спецификаций, планов и документации.

## Структура

```
apps/{app-name}/.specs/
├── README.md              # Описание структуры документации
├── spec.md                # Спецификация функциональности (или {APP}_PLAN.md)
├── plan.md                # Технический план (или IMPLEMENTATION_PLAN.md)
├── tasks.md               # Детальные задачи (опционально, может быть в корне)
├── analysis.md            # Анализ согласованности
├── SPECKIT_WORKFLOW.md    # Инструкция по командам speckit
└── checklists/
    └── {domain}.md        # Чек-листы качества требований
```

## Использование

1. Скопировать структуру в `apps/{app-name}/.specs/`
2. Заполнить README.md с описанием документации
3. Использовать команды speckit для генерации документов
4. Все документы автоматически сохраняются в `.specs/`

## Преимущества

- ✅ Все спецификации в одном месте
- ✅ Легко найти документацию для конкретного приложения
- ✅ Не засоряет корень проекта
- ✅ Единая структура для всех приложений
- ✅ Удобная навигация
